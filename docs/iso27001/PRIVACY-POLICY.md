## 서문

**암페놀센싱코리아**(이하 "회사")는 임직원의 자유와 권리 보호를 위해 「개인정보 보호법」 및 관계 법령이 정한 바를 준수하여, 적법하게 개인정보를 처리하고 안전하게 관리하고 있습니다. 이에 「개인정보 보호법」 제30조에 따라 정보주체에게 개인정보의 처리와 보호에 관한 절차 및 기준을 안내하고, 이와 관련한 고충을 신속하고 원활하게 처리할 수 있도록 하기 위하여 다음과 같이 개인정보 처리방침을 수립·공개합니다.

> 본 처리방침은 사내 그룹웨어 시스템(인사관리·전자결재·게시판·예산관리·캘린더·주소록·AI 챗봇 등)에 한하여 적용되며, 회사 전반의 임직원 개인정보 처리정책과는 별도로 운영됩니다.

## 1. 개인정보의 처리 목적

회사는 다음의 목적을 위하여 개인정보를 처리합니다. 처리하고 있는 개인정보는 다음의 목적 외의 용도로는 이용되지 않으며, 이용 목적이 변경되는 경우에는 「개인정보 보호법」 제18조에 따라 별도의 동의를 받는 등 필요한 조치를 이행합니다.

| 구분 | 처리 목적 |
|------|----------|
| 계정 관리 | 회원가입 의사 확인, 본인 식별·인증, 회원자격 유지·관리, 부정이용 방지 |
| 인사·근태 | 직원 정보 조회·관리, 부서 배치, 조직도 운영 |
| 협업 도구 | 게시판, 캘린더, 주소록, 프로젝트 관리 서비스 제공 |
| 전자결재 | 결재 문서 작성·승인·반려, 결재선 관리, 진행 상황 알림 |
| 예산관리(AR) | 프로젝트 예산 집행, 지출 등록·승인, 활동 이력 관리 |
| AI 서비스 | 사내 규정 챗봇 응답 생성, 전표 자동화 인식 보조 |
| 알림 | 결재·게시글·댓글·피드백·예산 경고 등 인앱·이메일·데스크탑 앱 알림 |
| 보안 관리 | 접근 통제, 로그인 이상 탐지, 감사 로그, 비밀번호 이력 관리 |
| 시스템 운영 | 장애 대응, 성능 분석, 서비스 개선 |

### 2.1 정보주체의 동의 없이 처리하는 항목 (계약 이행·법령상 의무·정당한 이익)

| 처리 목적 | 법적 근거 | 처리 항목 |
|----------|----------|----------|
| 계정 관리·인증 | 보호법 제15조제1항제4호 (계약 이행) | 아이디, 비밀번호(해시), 이름, 이메일, 사번, 권한(role), 계정 활성화 상태 |
| 인사·근태 관리 | 보호법 제15조제1항제2호 (법령상 의무 - 근로기준법) | 사번, 이름, 부서, 직급, 입사일, 내선번호, 휴대전화, 프로필 이미지, 부서장 여부 |
| 전자결재·예산 | 보호법 제15조제1항제4호 (계약 이행) | 결재자/기안자 정보, 부서, 직급, 결재 내역, 지출 내역 |
| 협업·게시판 | 보호법 제15조제1항제4호 (계약 이행) | 작성자명, 부서, 작성·수정 일시, 댓글, 첨부파일 |
| 보안·감사 | 보호법 제15조제1항제6호 (정당한 이익) | 로그인 IP, User-Agent, 접속 일시, 로그인 실패 횟수, 비밀번호 이력(bcrypt 해시) |
| 시스템 로그 | 보호법 제15조제1항제6호 (정당한 이익) | 사용자 ID, IP 주소, 요청 URL, 오류 메시지, 발생 일시 |
| 감사 로그 | 보호법 제15조제1항제2호 (법령상 의무 - 정보통신망법) | 접근자, 접근 대상 데이터, 접근 일시, 접근 사유 |

### 2.2 정보주체의 동의를 받아 처리하는 항목

| 처리 목적 | 법적 근거 | 처리 항목 |
|----------|----------|----------|
| 개인 주소록 | 보호법 제15조제1항제1호 (동의) | 외부 연락처(이름, 회사, 부서, 직급, 전화, 이메일, 메모) |
| 프로필 이미지 변경 | 보호법 제15조제1항제1호 (동의) | 사용자 업로드 이미지 |

### 2.3 자동으로 생성·수집되는 정보

| 항목 | 수집 방식 | 용도 |
|------|----------|------|
| 세션 쿠키 (`accessToken`, `refreshToken`) | 로그인 시 httpOnly 쿠키 발급 | 인증 유지, 자동 로그인 갱신 |
| 접속 로그 | 서버 미들웨어 자동 기록 | 보안 감사, 장애 분석 |
| 클라이언트 오류 정보 | `window.onerror` / React ErrorBoundary | 서비스 안정성 개선 |
| 게시글 열람 기록 (`post_views`) | 게시글 조회 시 자동 기록 | 미확인 공지 표시 |

### 2.4 민감정보 및 고유식별정보

회사는 그룹웨어 시스템에서 **주민등록번호·여권번호·운전면허번호·외국인등록번호 등 고유식별정보를 처리하지 않습니다.** 또한 사상·신념, 노동조합·정당 가입, 정치적 견해, 건강, 성생활 등의 **민감정보를 처리하지 않습니다.**

> 단, 회사 전반의 인사 시스템(별도 운영)에서는 법정 의무에 따라 일부 고유식별정보를 처리할 수 있으며, 이는 본 처리방침의 적용 범위를 벗어납니다.

## 3. 개인정보의 처리 및 보유 기간

회사는 법령에 따른 개인정보 보유·이용 기간 또는 정보주체로부터 개인정보를 수집 시에 동의받은 개인정보 보유·이용 기간 내에서 개인정보를 처리·보유합니다.

| 구분 | 보유 기간 | 근거 |
|------|----------|------|
| 임직원 계정 정보 | 재직 기간 + **퇴직 후 1년** (관련 분쟁·이의 처리용) | 회사 인사 규정 |
| 전자결재 문서 | **10년** (회계·세무 관련) | 「상법」 제33조, 「국세기본법」 제85조의3 |
| 예산관리(AR) 지출 내역 | **5년** | 「전자상거래법 시행령」 제6조 |
| 게시판·캘린더·프로젝트 | 작성자 퇴직 시까지 (업무 인수인계 후 익명화) | 회사 업무 규정 |
| 시스템 로그 | **90일** (자동 삭제) | `server.js scheduleDailyCleanup` |
| 감사 로그 (`audit_logs`) | **3년** | 정보통신망법 시행령 제15조 준용 |
| 비밀번호 이력 | 최근 **10개** 보관, 재사용 차단 시 최근 **3개** 검사 | 「개인정보의 안전성 확보조치 기준」 |
| 로그인 실패·잠금 기록 | 잠금 해제 후 **30일** | 보안 정책 |
| Refresh Token | 발급 후 **7일**, 만료 후 1시간마다 자동 삭제 | 인증 정책 |
| 개인 주소록 | 동의 철회 또는 회원 탈퇴 시까지 | 정보주체 동의 |

## 4. 개인정보의 파기 절차 및 방법

### 4.1 파기 절차
- 개인정보 보유 기간이 경과하거나 처리 목적이 달성된 경우, **지체 없이** 해당 개인정보를 파기합니다.
- 다른 법령에 따라 보존이 필요한 경우, 별도의 데이터베이스(또는 파일)로 분리 저장하여 법정 보존기간 동안만 보관합니다.
- 파기 사유가 발생한 개인정보는 개인정보 보호책임자의 승인을 받아 파기합니다.

### 4.2 파기 방법
| 매체 | 방법 |
|------|------|
| 전자적 파일 (DB 레코드) | `DELETE` 쿼리 + 트랜잭션 로그 정리, 백업 본도 보존기간 경과 후 폐기 |
| 첨부파일 (`uploads/`) | 운영체제 단계 영구 삭제 (`unlink`), 백업 매체에서도 동일 처리 |
| 종이 출력물 (관리자 보고서 등) | 분쇄기로 분쇄 또는 소각 |

### 4.3 자동 파기 작업
- `refresh_tokens` 만료 항목: **1시간**마다 자동 삭제 (`server.js setInterval`)
- `system_logs` 90일 이상 항목: 매일 **새벽 3시** 자동 삭제 (`scheduleDailyCleanup`)

## 5. 개인정보의 제3자 제공

회사는 정보주체의 개인정보를 본 처리방침에서 명시한 처리 목적 범위 내에서만 처리하며, **외부 제3자에게 개인정보를 제공하지 않습니다.**

다만, 다음의 경우에는 예외적으로 제공할 수 있습니다:
- 정보주체로부터 별도의 동의를 받은 경우
- 법령에 특별한 규정이 있는 경우 (수사·재판 영장, 세무조사 등)
- 정보주체 또는 제3자의 급박한 생명·신체·재산상 이익을 위해 필요한 경우

## 6. 개인정보 처리업무의 위탁

회사는 원활한 서비스 제공을 위해 다음과 같이 개인정보 처리 업무를 위탁하고 있습니다.

| 수탁자 | 위탁 업무 | 위탁 항목 | 보유·이용 기간 |
|--------|----------|----------|---------------|
| **Microsoft Corporation** (Office 365 SMTP) | 결재 알림·시스템 알림 메일 발송 | 수신자 이메일, 발신 메일 본문, 발송 시각 | 발송 후 즉시 처리, Microsoft 보존 정책에 따름 |
| **사내 RAG 서비스** (`rag_service`, Python FastAPI) | AI 챗봇 응답 생성 | 사용자 질문 텍스트, 사용자 ID | 질문 처리 직후 즉시 폐기 (저장 안 함) |
| **사내 Ollama 서버** | AI 전표 자동화 (PDF 텍스트 추출) | 업로드된 전표 PDF 본문 텍스트 | 처리 직후 메모리에서 제거 |

위탁 계약 시 「개인정보 보호법」 제26조에 따라 위탁업무 수행 목적 외 개인정보 처리 금지, 기술적·관리적 보호조치, 재위탁 제한, 수탁자에 대한 관리·감독, 손해배상 등 책임에 관한 사항을 계약서 등 문서에 명시하고 있습니다.

> Microsoft Office 365는 표준 기업 계약(MS Online Service Terms, Data Protection Addendum)을 통해 개인정보 보호 의무가 적용됩니다. RAG 서비스와 Ollama는 사내 인프라에서 운영되어 외부로 데이터가 전송되지 않습니다.

## 8. 개인정보의 안전성 확보조치

회사는 「개인정보의 안전성 확보조치 기준」(개인정보보호위원회 고시 제2025-9호) 및 ISO 27001 통제에 따라 다음과 같은 조치를 취하고 있습니다.

### 8.1 관리적 조치
- 내부 관리계획 수립·시행 (`SECURITY-POLICY.md`)
- 개인정보 취급자 최소화 및 권한 등급 분리 (USER / DEPT_ADMIN / HR_ADMIN / SUPER_ADMIN)
- 정기적 직원 보안 교육 및 비밀번호 변경 안내
- **반기별 접근권한 검토** (`/admin/access-review`)
- 보안 인시던트 대응 절차 (`INCIDENT-RESPONSE.md`)

### 8.2 기술적 조치
- 개인정보처리시스템 접근 통제 (역할 기반 권한, JWT 토큰 인증)
- 비밀번호 단방향 암호화 (bcrypt, salt round 12)
- 비밀번호 이력 관리 (최근 3개 재사용 차단, `password_history` 테이블)
- 통신 구간 암호화 (HTTPS/TLS, 외부 접속 시)
- 접속 기록 보관 및 점검 (`system_logs`, `audit_logs`)
- 보안 헤더 적용 (Helmet, CSP, X-Frame-Options 등)

## 10. 정보주체와 법정대리인의 권리·의무 및 행사방법

정보주체는 회사에 대해 언제든지 다음의 권리를 행사할 수 있습니다.

| 권리 | 행사 방법 |
|------|----------|
| 개인정보 **열람** | `내 정보` 메뉴에서 직접 조회, 또는 개인정보 보호책임자에게 서면·이메일 요청 |
| 개인정보 **정정** | `내 정보` 메뉴에서 직접 수정 (이름·연락처 등), 사번·부서 변경은 인사 담당자 통해 요청 |
| 개인정보 **삭제** | 퇴직 시 자동 비활성화 후 보유 기간 경과 시 삭제, 그 외 삭제 요구는 보호책임자에게 요청 |
| 개인정보 **처리정지** | 보호책임자에게 서면·이메일 요청 (단, 법령상 의무·계약 이행을 위해 거부될 수 있음) |
| 개인정보 **전송 요구** | 보호책임자에게 요청 시 표준 형식(JSON·CSV)으로 제공 |
| **동의 철회** | 개인 주소록 등 동의 기반 처리 항목은 해당 화면에서 즉시 삭제 가능 |
| 자동화된 결정 **거부·설명 요구** | 11항 참조 |

### 10.1 권리 행사 절차
1. 정보주체가 보호책임자에게 요청 (방문·서면·이메일·전화 모두 가능)
2. 본인 확인 후 **10일 이내** 처리 결과 회신 (전송요구는 지체 없이)
3. 거절 시 사유 명시

### 10.2 권리 행사의 제한
- 다른 법령에서 그 개인정보가 수집 대상으로 명시되어 있는 경우, 해당 개인정보의 삭제를 요구할 수 없습니다 (예: 결재 문서, 회계 기록).
- 「개인정보 보호법」 제35조제4항 및 제37조제2항에 따라 열람·처리정지 요구가 제한될 수 있습니다.

## 11. 자동화된 결정에 관한 사항

회사의 그룹웨어는 다음의 자동화된 처리를 수행합니다.

| 기능 | 자동화 처리 내용 | 정보주체에 미치는 영향 |
|------|----------------|---------------------|
| 로그인 실패 시 계정 잠금 | 5회 연속 실패 시 자동 잠금 | 일시적 서비스 이용 제한 (관리자 해제 필요) |
| AI 챗봇 응답 | 사용자 질문 → RAG 모델 → 응답 생성 | 정보 제공 (의사결정 보조, 자동 결정 아님) |
| AI 전표 자동화 | PDF 텍스트 → LLM 추출 → 전표 양식 채움 | 사용자가 최종 확인·수정 (자동 결정 아님) |
| 결재 자동 라우팅 | 결재선 규칙에 따라 다음 결재자 자동 지정 | 결재 절차 안내 (실제 승인은 사람이 수행) |

위 처리는 모두 **사람이 최종 의사결정을 하는 보조 도구**이며, 「개인정보 보호법」 제37조의2에서 정한 "정보주체의 권리·의무에 중대한 영향을 미치는 자동화된 결정"에 해당하지 않습니다.

다만 정보주체가 위 처리에 대해 **거부·설명을 요구**할 경우, 보호책임자에게 요청하여 30일 이내 처리 결과를 받을 수 있습니다.
