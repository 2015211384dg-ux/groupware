# 변경관리 절차서

**문서번호:** CMP-001  
**버전:** 1.0  
**작성일:** 2026-04-24  
**ISO 27001 조항:** A.8.25, A.8.32  

---

## 1. 목적

그룹웨어 시스템의 소프트웨어·설정·인프라 변경이 보안 취약점, 데이터 유실, 서비스 중단 없이 안전하게 반영되도록 통제한다.

---

## 2. 적용 범위

| 변경 유형 | 예시 |
|-----------|------|
| 코드 변경 | 신규 기능, 버그 수정, 리팩토링 |
| 설정 변경 | .env, ecosystem.config.js, DB 스키마 |
| 인프라 변경 | Node.js 버전 업그레이드, MariaDB 패치, OS 업데이트 |
| 권한 변경 | 사용자 역할 변경, DB 계정 권한 조정 |

---

## 3. 변경 분류

| 등급 | 정의 | 승인 | 사전 공지 |
|------|------|------|----------|
| **긴급(Emergency)** | 보안 취약점 패치, 서비스 장애 복구 | IT 관리자 단독 | 사후 1시간 내 공지 |
| **일반(Normal)** | 신규 기능, 기존 기능 수정 | IT 관리자 검토 후 배포 | 배포 1일 전 |
| **대규모(Major)** | DB 스키마 변경, Node.js 버전 업, 마이그레이션 | IT 관리자 + 경영진 | 배포 3일 전, 점검 공지 |

---

## 4. 일반 변경 절차

```
1. 개발 (개발 PC, groupware_dev DB)
   └─ 기능 구현 및 로컬 테스트
   
2. 검토
   └─ 코드 보안 검토 (OWASP Top 10 체크)
   └─ DB 스키마 변경 시 롤백 스크립트 준비

3. 스테이징 테스트 (개발 PC에서 운영 데이터 복제본으로 확인)
   └─ DEV-WORKFLOW.md 절차 참조

4. 배포 승인
   └─ IT 관리자 확인 후 GitHub main 브랜치 머지

5. 운영 배포
   └─ 운영 서버에서 git pull origin main
   └─ npm install (패키지 변경 시)
   └─ npm run build (프론트엔드 변경 시)
   └─ pm2 restart groupware-backend --update-env

6. 배포 확인
   └─ pm2 status 정상 확인
   └─ 주요 기능 동작 확인
   └─ pm2 logs --lines 20 오류 없음 확인

7. 기록
   └─ Git 커밋 메시지에 변경 내용 명시
   └─ 중요 변경은 아래 §6 변경 이력에 기록
```

---

## 5. DB 스키마 변경 절차

DB 스키마 변경(테이블 추가/수정/삭제)은 추가 주의가 필요:

```
1. 변경 전 전체 DB 백업
   mysqldump --defaults-file=C:\groupware\backup\my-backup.cnf \
     --single-transaction --host=127.0.0.1 --port=3300 \
     groupware > D:\Backup\groupware\pre-change-YYYYMMDD.sql

2. 롤백 스크립트 준비 (예: DROP TABLE IF EXISTS ...)

3. server.js의 CREATE TABLE IF NOT EXISTS 패턴 사용
   (운영 서버 재시작 시 자동 생성, 기존 테이블 영향 없음)

4. 배포 후 데이터 정합성 확인
   SELECT COUNT(*) FROM 새테이블;
```

## 6. 긴급 변경 절차

보안 취약점 발견 또는 서비스 장애 시:

```
1. 즉시 조치 (점검 모드 ON → 수정 → 재배포)
   - 관리자 설정 > 점검 모드 활성화
   
2. 수정 및 배포 (일반 절차 3~6단계)

3. 사후 보고 (1시간 이내)
   - 변경 내용, 영향 범위, 원인 분석을 §6에 기록
   - 필요 시 직원 공지 (보안 사고인 경우 IRP-001 병행)
```

## 8. 보안 검토 체크리스트

배포 전 다음 항목을 확인:

- [ ] SQL Injection 방지 (Parameterized Query 사용)
- [ ] XSS 방지 (사용자 입력 이스케이프)
- [ ] 인증 확인 (authMiddleware 적용 여부)
- [ ] 권한 확인 (checkRole 적용 여부)
- [ ] 파일 업로드 시 MIME/확장자 검증
- [ ] .env에 민감정보 하드코딩 없음
- [ ] console.log에 민감정보 출력 없음
- [ ] npm audit 이상 없음
