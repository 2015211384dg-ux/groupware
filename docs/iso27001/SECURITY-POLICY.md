# 정보보안 정책서

**문서번호:** ISP-001  
**버전:** 1.0  
**작성일:** 2026-04-24  
**적용범위:** 암페놀센싱코리아 그룹웨어 시스템  
**시스템 URL:** http://10.18.10.70:3000  

---

## 1. 목적

본 정책은 그룹웨어 시스템에서 처리되는 정보자산(직원 개인정보, 결재문서, 재무데이터, 첨부파일 등)을 보호하고, ISO 27001 요구사항을 준수하기 위한 기준을 정의한다.

## 2. 적용 범위

| 대상 | 내용 |
|------|------|
| 시스템 | 그룹웨어 서버 (10.18.10.70), MariaDB (port 3300), 데스크탑 앱 (Electron) |

## 3. 정보자산 분류

| 등급 | 정의 | 해당 데이터 | 접근 제한 |
|------|------|-----------|-----------|
| **기밀 (Confidential)** | 외부 유출 시 심각한 피해 | 인사정보(급여·평가), AR 재무데이터, 결재문서 | 권한 보유자만 |
| **내부 (Internal)** | 사내 임직원만 열람 | 게시판, 캘린더, 주소록, 프로젝트 | 전 임직원 |
| **공개 (Public)** | 외부 공개 무방 | 공지사항(지정 시) | 제한 없음 |

## 4. 역할 및 접근 권한

| 역할 | 코드 | 주요 권한 |
|------|------|-----------|
| 최고관리자 | `SUPER_ADMIN` | 모든 기능, 시스템 설정, 사용자 관리, 계정 잠금 해제 |
| 관리자 | `ADMIN` | 게시판·결재 관리, 사용자 조회/수정, 보안 알림 수신 |
| 인사관리자 | `HR_ADMIN` | 인사정보 조회·수정, 직원 관리 |
| 일반사용자 | `USER` | 본인 관련 기능, 공개 게시판, 전자결재 |

**원칙:**
- 최소권한 원칙: 업무에 필요한 최소 권한만 부여
- 직무분리: SUPER_ADMIN 계정은 일상 업무용으로 사용 금지, 관리 작업 전용
- 퇴직/전보 즉시 계정 비활성화 (`is_active = FALSE`)

## 5. 인증 정책

### 5.1 비밀번호 정책
| 항목 | 기준 | 설정 위치 |
|------|------|-----------|
| 최소 길이 | 8자 이상 | 설정 > 시스템 설정 |
| 특수문자 | 포함 권장 (설정으로 필수화 가능) | 설정 > 시스템 설정 |
| 초기 비밀번호 | 최초 로그인 시 강제 변경 | `require_password_change = 1` |
| 비밀번호 재사용 | 권고: 직전 3개 미사용 | 수동 관리 |

### 5.2 계정 잠금 정책
- 비밀번호 연속 실패 **5회** → 30분 자동 잠금
- 잠금 발생 시 관리자에게 **즉시 인앱 알림** 발송
- 잠금 해제: SUPER_ADMIN이 관리자 페이지에서 수동 해제

### 5.3 세션 정책
- 액세스 토큰 유효기간: **15분** (자동 갱신)
- 리프레시 토큰 유효기간: **7일**
- 유휴 세션 자동 로그아웃: **60분** (관리자 설정 변경 가능)
- 로그아웃 시 서버 측 토큰 즉시 폐기

## 6. 접근통제 정책

### 6.1 네트워크 접근
- 그룹웨어는 **내부망(10.18.10.x)** 전용 운영
- 외부 인터넷에서의 직접 접속 불가 (방화벽 차단)
- VPN을 통한 원격 접속 시 동일한 인증 정책 적용

### 6.2 파일 접근
- 업로드 파일(`/uploads/`)은 인증된 사용자만 접근 가능
- 파일 업로드 시 확장자 + 매직바이트(파일 실제 형식) 이중 검증
- 허용 확장자: jpg/jpeg/png/gif/webp/bmp/pdf/doc/docx/xls/xlsx/ppt/pptx/hwp/zip/7z/rar/txt/csv/log/rtf
- 파일 크기 제한: 게시판 10MB, 결재문서 20MB

### 6.3 API 접근
- 모든 API 엔드포인트는 JWT 인증 필수 (공개 엔드포인트 예외: `/auth/login`, `/settings/public`, `/settings/maintenance`)
- Rate Limiting: 15분당 3,000회 (내부망 제외)
- CORS: 허용 Origin 명시 (http://10.18.10.70:3000)

## 7. 암호화 정책

| 대상 | 방식 |
|------|------|
| 비밀번호 | bcrypt (salt round 10) |
| JWT | HS256, 128자 랜덤 시크릿 |
| 리프레시 토큰 | SHA-256 해시 후 DB 저장 |
| 통신 구간 | HTTP (내부망) — HTTPS 전환 시 TLS 1.2 이상 적용 |
| DB 접속 | 전용 계정(`groupware_app`) 사용, root 계정 미사용 |

## 8. 로그 및 감사

| 로그 종류 | 테이블 | 보관 기간 | 기록 내용 |
|-----------|--------|-----------|-----------|
| 시스템 활동 로그 | `system_logs` | 90일 | 로그인/로그아웃, 계정 잠금, 주요 변경사항 |
| 감사 로그 | `audit_logs` | 1년 | 결재문서 열람, 민감 데이터 접근 |
| 결재 히스토리 | `approval_documents` | 영구 | 결재선, 승인/반려 이력 |
| AR 활동 로그 | `ar_activity_logs` | 영구 | 예산 변경, 지출 등록 이력 |

**로그 접근:** SUPER_ADMIN만 직접 조회 가능  
**로그 무결성:** DB 접근 권한 제한으로 보호 (일반 사용자 DELETE/TRUNCATE 권한 없음)
